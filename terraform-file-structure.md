# Terraform File Structure Standard

> **Status:** Active — this standard is in effect for all Terraform module and root configuration development.

---

## Policy Statement

All Terraform modules and root configurations separate resources by purpose into dedicated files. Mixing resource types or concerns in a single file is not permitted. Root configurations contain module calls only — zero inline `resource` blocks.

---

## Repository Layout

│   ├── avm-wrappers/                   # Thin wrappers over AVM modules (temporary)

│   │       ├── outputs.tf
│   │       ├── versions.tf
│   │       └── README.md
│   └── custom/                         # AVM-pattern custom modules
│       └── <resource-type>/
│           ├── main.tf
│           ├── variables.tf
│           ├── outputs.tf

## File Separation Rules

| File | Contents | Rules |
|---|---|---|
| `main.tf` | Primary resource block(s) for the module's core resource. Root configs: module call blocks only. | No mixed resource types. Root configs: zero `resource` blocks. |
| `variables.tf` | All `variable` declarations | One file — no variables declared elsewhere |
| `outputs.tf` | All `output` declarations | One file — no outputs declared elsewhere |
| `versions.tf` | `terraform {}` block, `required_providers`, version constraints | No resource or variable blocks |
| `locals.tf` | `locals {}` blocks only | No resource or data blocks |
| `data.tf` | `data` source lookups only | No resource blocks |
| `private_endpoints.tf` | All `azurerm_private_endpoint` and `azurerm_private_dns_zone_group` resources | Always a separate file — makes PE config auditable |
| `diagnostic_settings.tf` | All `azurerm_monitor_diagnostic_setting` resources | Always a separate file |
| `role_assignments.tf` | All `azurerm_role_assignment` resources | Always a separate file |
| `locks.tf` | All `azurerm_management_lock` resources | Always a separate file |
| `backend.tf` | `backend {}` block only | Root configurations only |

---

## Root Configuration Rules

- Root configurations (`workloads/`, `platform/`) contain **module calls only**
- All resource definitions live inside modules — never inline in root configs
- Root configurations wire module outputs to module inputs
- Environment-specific values supplied exclusively via `.tfvars` files — never hardcoded
- Secrets are never in `.tfvars` files — always referenced from Key Vault

---

## Naming Convention (File Level)

- All files use lowercase with underscores: `private_endpoints.tf`, `role_assignments.tf`
- Module directories use lowercase with hyphens: `key-vault/`, `storage-account/`
- `.tfvars` files named by environment: `dev.tfvars`, `staging.tfvars`, `prod.tfvars`

---

## Documentation

- Every module includes a `README.md` auto-generated by `terraform-docs`
- Manual README authoring is not permitted — all documentation is derived from variable and output descriptions
- Run `terraform-docs` as part of the PR pipeline gate
